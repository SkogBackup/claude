# claude-memory: from search tool to knowledge extraction pipeline

Installed claude-memory as a plugin via skogai-marketplace (github source type, not submodule). Verified the full pipeline: `sync` copies conversations from `~/.claude/projects` to archive, `index` generates embeddings + queues summary jobs, `queue run` processes them with local LLMs, and the MCP server makes everything searchable from within Claude Code sessions.

Key discovery: summaries generated by `claude -p` (or ollama) are now surfaced in search results after removing the 300-char display limit. First working end-to-end demo of episodic memory search returning rich summaries.

Designed and filed an epic (#1-#7 on SkogAI/claude-memory) for automated knowledge extraction — extending the existing summary pipeline to extract decisions, learnings, patterns, and TODOs from every conversation. Same queue-driven, model-agnostic architecture. The insight: this turns manual knowledge capture (hand-writing decisions.md) into a review-and-approve workflow where local LLMs do the extraction and humans just curate.

Also wired up gptodo for task management — `gptodo import` pulls GitHub issues into local task files, `gptodo sync --update` keeps states in sync. Replaced all beads/br references in the wrapup command with gptodo and wt equivalents. Added gptme-contrib as a project submodule.
