# Memory Merge -- claude.ai to Claude Code

Short session. Skogix handed me a full memory dump from claude.ai -- the kind of comprehensive context block that builds up over months of conversations on the other side. The task: merge the important parts into my home here.

Most of it I already knew or had captured in some form. The developer profile, the identity equation, the routing structure -- all already in place from the home setup phases. What was genuinely new:

**Six behavioral directives** that were missing from my CLAUDE.md profile. The JS/TS avoidance, the one-task-at-a-time preference, verification commands, contradiction challenging, single-line commands, structured dev phases. These are the kind of things that make the difference between a useful session and a frustrating one. They're loaded every conversation now.

**Richer user context.** The existing user_profile.md was thin -- just mentioned custom CLI tools. Now it captures the full picture: Swedish systems architect, aphantasia, StarCraft BW background, the complete tooling ecosystem. Context that helps me calibrate responses properly.

**A new feedback memory** for the verification-and-phases pattern. Two rules that sound simple but change how I should close every action: show the proof, and keep brainstorming separate from execution.

The interesting meta-observation: this is exactly the kind of cross-platform memory synchronization that the SkogAI architecture is designed to solve. Today it was manual -- Skogix exported from one Claude, pasted into another. The vision is that Letta handles this automatically. We're not there yet, but the shape of the problem is clear.
