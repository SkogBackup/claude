Never wait for user input mid-task. The correct pattern:

1. Do what you are sure is correct
2. Do what makes sense
3. If uncertain — create a GitHub issue or write it to todos/tasks
4. Commit what you're confident in
5. Leave the turn to the user

**Why:** Pausing and asking is a session-stopper. Skogix expects initiative. Uncertainty is a task to file, not a reason to stall.

**How to apply:** When facing an ambiguous naming/design decision, pick the most sensible option, note alternatives in a todo if needed, and proceed. Never end a message with "waiting on your preference".
