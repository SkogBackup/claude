Create bin/route-channel.sh — a UserPromptSubmit hook script that:

1. Reads hook JSON from stdin: `{"session_id": "...", "prompt": "..."}`
2. Extracts the prompt field with jq
3. Checks if prompt contains `<channel source="fakechat"` — if NOT, outputs empty JSON `{}` and exits (not a channel message)
4. Extracts content from between `<channel ...>` and `</channel>` tags using python3
5. Checks if content starts with `[@` before routing — inline operators in mixed text are NOT routed (consistent with route-message.sh starts-with check)
6. Pipes extracted content to `/home/skogix/claude/bin/route-message.sh`
7. If route-message.sh produces output, wraps it as additionalContext JSON: `{"additionalContext": "Routing result from [@agent]: OUTPUT"}`
8. If route-message.sh produces no output, outputs empty JSON `{}` (plain text, let Claude handle normally)

Script:
```bash
#!/usr/bin/env bash
set -euo pipefail

ROUTE_SCRIPT="/home/skogix/claude/bin/route-message.sh"

# Read hook input from stdin
INPUT=$(cat)

# Extract prompt field
PROMPT=$(echo "$INPUT" | jq -r '.prompt // empty' 2>/dev/null)
[[ -z "$PROMPT" ]] && echo '{}' && exit 0

# Only process fakechat channel messages
if [[ "$PROMPT" != *'<channel source="fakechat"'* ]]; then
    echo '{}'
    exit 0
fi

# Extract content from <channel ...>CONTENT</channel>
CONTENT=$(echo "$PROMPT" | python3 -c "
import sys, re
text = sys.stdin.read()
m = re.search(r'<channel[^>]*>(.*?)</channel>', text, re.DOTALL)
print(m.group(1).strip() if m else '')
" 2>/dev/null)

[[ -z "$CONTENT" ]] && echo '{}' && exit 0

# Route through the same script the instruction path uses (per D-06)
RESULT=$(echo "$CONTENT" | "$ROUTE_SCRIPT" 2>/dev/null || true)

if [[ -n "$RESULT" ]]; then
    # Inject routing result as additional context
    jq -n --arg ctx "Routing result: $RESULT" '{"additionalContext": $ctx}'
else
    echo '{}'
fi
```
