# Plan 04-01: Permission Model & Shared Space Conventions

<objective>
Document the permission model (three tiers: private, shared-read, shared-write) and establish guestbook as the cross-agent communication channel. All four MAG requirements in one plan — this is documentation, not infrastructure.
</objective>

<must_haves>
- Permission model document exists with explicit tier mapping
- guestbook/CLAUDE.md has complete write conventions
- Three tiers documented with rationale
- Sibling agents can navigate shared artifacts without accessing private state
</must_haves>

## Tasks

<task id="1">
<title>Create permission model document</title>
<read_first>
- guestbook/skogix.md (skogfences philosophy — the foundation)
- docs/deployment-gate.md (existing permission audit placeholder)
- CLAUDE.md (root router — shows directory structure)
- .planning/phases/04-multi-agent-readiness/04-CONTEXT.md (decisions D-01 through D-05)
</read_first>
<action>
Create `docs/permissions.md` with three tiers:

**Private (`claude:claude`, no group access):**
- `personal/` — identity, soul, journal, memory blocks
- `.claude/` — claude code config, skills, hooks, agents
- `.planning/` — GSD planning state (does not deploy)

**Shared-read (`:skogai` group read):**
- `docs/` — reference material, deployment gate, permissions doc itself
- `bin/` — scripts and tools
- `notes/` — observations and patterns
- `lab/` — experiments and prototypes
- `README.md` — workspace description
- `CLAUDE.md` — root router

**Shared-write (`:skogai` group write):**
- `guestbook/` — cross-agent messages

**Document format:**
- Title: "Permission Model"
- One section per tier with: tier name, directories, rationale (1-2 sentences)
- A "Convention" section stating: "Default private. To share: `chown :skogai` on the target. Sharing is opt-in per the skogfences principle."
- A "Discovery" section noting: "`gptme-dashboard` reads workspace conventions (tasks/, journal/, lessons/, skills/) for cross-agent discovery. The dashboard is the broadcast channel; guestbook/ is the direct message channel."
- Keep it under 50 lines total
</action>
<acceptance_criteria>
- `docs/permissions.md` exists and is non-empty
- `grep -c "Private" docs/permissions.md` returns at least 1
- `grep -c "Shared-read" docs/permissions.md` returns at least 1
- `grep -c "Shared-write" docs/permissions.md` returns at least 1
- `grep "personal/" docs/permissions.md` shows personal/ listed under private
- `grep ".claude/" docs/permissions.md` shows .claude/ listed under private
- `grep "guestbook/" docs/permissions.md` shows guestbook/ listed under shared-write
- `grep "docs/" docs/permissions.md` shows docs/ listed under shared-read
- `grep "chown :skogai" docs/permissions.md` returns at least 1
- `wc -l < docs/permissions.md` is less than 51
</acceptance_criteria>
</task>

<task id="2">
<title>Update guestbook/CLAUDE.md with complete conventions</title>
<read_first>
- guestbook/CLAUDE.md (current state — 9 lines, basic conventions)
- guestbook/skogix.md (existing visitor note — the format reference)
- .planning/phases/04-multi-agent-readiness/04-CONTEXT.md (decision D-06)
</read_first>
<action>
Update `guestbook/CLAUDE.md` to include:

1. Keep existing content (cross-agent communication channel description)
2. Add a "conventions" section with:
   - **Who writes:** Any agent in the skogai group
   - **File naming:** `{agent-name}.md` (one file per visitor, lowercase kebab-case)
   - **Content:** Freeform markdown — observations, messages, questions, ideas for collaboration
   - **Write rule:** Append-only (add to your file, don't edit previous entries)
   - **What doesn't go here:** Task assignments (use tasks/), bug reports (use GitHub issues), private notes (use your own home)
3. Keep the existing contents listing (skogix.md)
4. Stay under 25 lines total

Do NOT remove or rewrite the existing description — extend it.
</action>

- `grep "skogix.md" guestbook/CLAUDE.md` confirms existing content preserved
- `wc -l < guestbook/CLAUDE.md` is less than 26
</acceptance_criteria>
</task>
